Go 语言并发编程模式详解

编码 2025-12-18 · 0 次阅读 · 预计阅读 12 分钟

Go 语言诞生于 2009 年，由 Robert Griesemer、Rob Pike 和 Ken Thompson 设计。它的并发模型基于 CSP（Communicating Sequential Processes） 理论，通过 Goroutine 和 Channel 提供了简洁而强大的并发编程能力。

不同于传统的线程加锁模型，Go 倡导"不通过共享内存来通信，而通过通信来共享内存"。这一理念催生了许多优雅的并发编程模式。

Goroutine 基础

Goroutine 是 Go 运行时管理的轻量级线程，创建成本极低（栈初始仅几 KB）。只需在函数调用前加上 go 关键字即可启动一个 Goroutine：

go func() {
    fmt.Println("Hello from goroutine")
}()

与系统线程不同，Goroutine 的调度由 Go 运行时负责（M:N 调度模型），这使得我们可以轻松创建成千上万个 Goroutine。

模式一：扇出/扇入（Fan-Out/Fan-In）

扇出模式是指将同一个任务分发到多个 Goroutine 并行处理，扇入模式则是将多个 Goroutine 的结果合并到一个 Channel 中。

// 扇出：将数据分发到多个 worker
func fanOut(input <-chan int, workers int) []<-chan int {
    channels := make([]<-chan int, workers)
    for i := 0; i < workers; i++ {
        channels[i] = worker(input, i)
    }
    return channels
}

func worker(input <-chan int, id int) <-chan int {
    output := make(chan int)
    go func() {
        defer close(output)
        for val := range input {
            output <- val * 2 // 模拟处理
        }
    }()
    return output
}

// 扇入：合并多个 channel 为一个
func fanIn(channels ...<-chan int) <-chan int {
    out := make(chan int)
    var wg sync.WaitGroup
    for _, ch := range channels {
        wg.Add(1)
        go func(c <-chan int) {
            defer wg.Done()
            for val := range c {
                out <- val
            }
        }(ch)
    }
    go func() {
        wg.Wait()
        close(out)
    }()
    return out
}

这种模式在处理大量独立任务时非常有用。比如批量处理 HTTP 请求、并行计算等场景。

模式二：Pipeline（流水线）

Pipeline 模式将数据处理流程划分为多个阶段，每个阶段由一组 Goroutine 处理，通过 Channel 连接：

// 阶段 1：生成数字
func generate(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for _, n := range nums {
            out <- n
        }
    }()
    return out
}

// 阶段 2：平方
func square(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            out <- n * n
        }
    }()
    return out
}

// 阶段 3：乘以 2
func multiply(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            out <- n * 2
        }
    }()
    return out
}

// 使用
func main() {
    result := multiply(square(generate(1, 2, 3, 4)))
    for v := range result {
        fmt.Println(v) // 2, 8, 18, 32
    }
}

Pipeline 的优雅之处在于每个阶段都可以独立并发执行，且可以轻松添加新的处理阶段而无需修改已有代码。

模式三：超时控制

在生产环境中，我们需要防止 Goroutine 无限期阻塞。通过 select 配合 time.After 可以实现超时控制：

func doWithTimeout(timeout time.Duration) error {
        result := make(chan error, 1)

        go func() {
                // 执行耗时操作
                result <- doSomething()
        }()

        select {
        case err := <-result:
                return err
        case <-time.After(timeout):
                return fmt.Errorf("operation timed out after %v", timeout)
        }
}

注意这里使用了带缓冲的 Channel（容量为 1），防止 Goroutine 在超时后向 Channel 发送数据时永远阻塞。

模式四：工作池（Worker Pool）

工作池模式限制同时执行的 Goroutine 数量，防止资源耗尽：

func workerPool(jobs []Job, workers int) []Result {
        jobCh := make(chan Job, len(jobs))
        resultCh := make(chan Result, len(jobs))

        // 启动 worker
        var wg sync.WaitGroup
        for i := 0; i < workers; i++ {
                wg.Add(1)
                go func(id int) {
                        defer wg.Done()
                        for job := range jobCh {
                                resultCh <- processJob(job)
                        }
                }(i)
        }

        // 分发任务
        for _, job := range jobs {
                jobCh <- job
        }
        close(jobCh)

        // 等待完成
        go func() {
                wg.Wait()
                close(resultCh)
        }()

        // 收集结果
        var results []Result
        for r := range resultCh {
                results = append(results, r)
        }
        return results
}

工作池的核心优势在于控制并发度。通常将 worker 数量设置为 runtime.NumCPU() 或根据 I/O 瓶颈调整。

模式五：Context 传递与取消

Go 1.7 引入的 context 包为 Goroutine 提供了取消信号和超时传递的标准方式：

func handleRequest(ctx context.Context, data []byte) error {
        ctx, cancel := context.WithTimeout(ctx, 5*time.Second)
        defer cancel()

        resultCh := make(chan error, 1)

        go func() {
                resultCh <- processData(ctx, data)
        }()

        select {
        case err := <-resultCh:
                return err
        case <-ctx.Done():
                return ctx.Err()
        }
}

func processData(ctx context.Context, data []byte) error {
        select {
        case <-time.After(3 * time.Second):
                return nil
        case <-ctx.Done():
                return ctx.Err()
        }
}

使用 Context 的关键原则：

  • 将 Context 作为函数的第一个参数传递
  • 不要将 Context 存储在结构体中
  • 及时调用 cancel 函数释放资源

总结

Go 的并发模型在处理现代分布式系统和网络服务时展现了极大的优势。熟练掌握这些并发模式，能帮助我们写出更高效、更健壮的 Go 程序。

在实际开发中，不必刻意使用并发。遵循"先做对，再优化"的原则，只在必要时引入并发，并始终考虑取消传播和资源管理。
